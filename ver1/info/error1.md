# Ошибки XML и форматов

## Ошибки XML

- Некорректная XML-разметка: браузерный `DOMParser` возвращает `parsererror`, текст ошибки выводится в нижнее окно лога.
- Пустой документ: XML не содержит корневой элемент.
- Незакрытый тег, неверная вложенность или неэкранированный символ `&`: ошибка отображается до рендеринга диаграммы.

## Ошибки распознавания формата

- BPMN определяется по корню `definitions` и namespace BPMN 2.0.
- draw.io определяется по корню `mxfile` или `mxGraphModel`.
- Archi определяется по корню `model` с namespace ArchiMate или атрибутом `identifier`.
- Если корневой элемент не соответствует этим правилам, сервис пишет в лог: `Формат не распознан`.

## Ошибки рендеринга

- Для BPMN используется `bpmn-js`; если XML валиден, но не содержит корректные BPMN DI-координаты, библиотека может вернуть ошибку импорта.
- Для draw.io и Archi используется универсальное SVG-представление ключевых элементов. Если элементы не найдены, выводится сообщение `Нет элементов для отображения`.
- Экспорт PNG/SVG доступен для SVG-представления. BPMN-диаграммы, отрисованные `bpmn-js`, можно сохранить через PDF-печать браузера.
