# Популярные on-line viewer и способы передачи схемы

## BPMN

- bpmn.io demo: основной web-viewer экосистемы `bpmn-js`. Подходит для BPMN 2.0 XML, чаще используется через вставку XML в интерфейс или через сжатые/закодированные параметры URL в демо-страницах.
- afomi.github.io/bpmn-viewer: простой viewer на базе `bpmn-js`. Удобен как пример передачи схемы параметром URL.
- bpmncat.ru: русскоязычный BPMN-инструмент, полезен как внешний сервис для ручного просмотра и проверки BPMN.
- Visual Paradigm Online: мощный online-редактор диаграмм. Для прямой передачи XML обычно требуется импорт через UI или внутренние параметры проекта.

## draw.io / diagrams.net

- app.diagrams.net: основной online-viewer/editor draw.io. Может открывать файлы `.drawio`, поддерживает URL-параметры и фрагмент `#R...` для закодированных данных, но формат параметров зависит от режима сжатия.
- viewer.diagrams.net: режим просмотра draw.io-диаграмм, удобен для публикации уже сохраненных файлов.

## Archi / ArchiMate

- ArchiMate Exchange File Format Validator и специализированные ArchiMate-viewer решения обычно принимают файл через форму загрузки.
- Универсального публичного viewer с устойчивым API передачи XML через URL меньше, чем для BPMN и draw.io. Поэтому в текущем сервисе Archi отображается через универсальную SVG-схему элементов.

## Дополнительные форматы для развития

- PlantUML: текстовый DSL, отлично передается через URL после deflate/base64-кодирования.
- Mermaid: текстовый DSL, хорошо подходит для GitHub Pages и может рендериться библиотекой `mermaid`.
- GraphML: XML-формат графов, полезен для yEd и сетевых схем.
- SVG: уже является XML и может отображаться напрямую в браузере.
- XMI/UML: применим для UML-моделей, но требует отдельной логики интерпретации.

## Методы передачи XML

- Query-параметр `?xml=` с base64-строкой. Используется в `xml-dia`, потому что работает на GitHub Pages и при локальном открытии.
- URL-фрагмент `#...`: меньше уходит на сервер, но GitHub Pages все равно обслуживает один и тот же файл.
- Внешняя ссылка на файл: компактно, но требует CORS и доступности исходного файла.
- Сжатый параметр URL: нужен для крупных схем, но усложняет совместимость без дополнительных библиотек.
